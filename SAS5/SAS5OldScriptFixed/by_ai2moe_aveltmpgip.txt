最该先改的地方
方案：统一做“旧地址 -> 新地址”的映射
这是最稳的。

你已经有 sentences 记录每段文本长度变化了。

做一个函数：

<CPP>
auto remapOldToNew = [&](uint32_t oldAddr) -> uint32_t {
    int delta = 0;
    for (auto& s : sentences) {
        if (s.addr < oldAddr)
            delta += s.offset;
    }
    return oldAddr + delta;
};
修绝对跳转
你现在的绝对跳转修法可以直接改成：

<CPP>

收集相对跳转
把注释去掉，但建议改成这样：

<CPP>
struct RelJmp {
    uint32_t addrInBuffer;     // 原文件里这4字节立即数的位置
    uint32_t addrInNewBuffer;  // 新文件里这4字节立即数的位置
};
case 0x10:
case 0x11:
case 0x12:
case 0x13: {
    int32_t rel = read<int32_t>(&buffer[offset]);
    uint32_t immAddrOld = offset;
    uint32_t immAddrNew = (uint32_t)newBuffer.size();
    offset += 4;
    // 先全部记录，后面统一修
    relJmps.push_back({ immAddrOld, immAddrNew });
    newBuffer.insert(newBuffer.end(),
        (uint8_t*)&rel, (uint8_t*)&rel + 4);
    break;
}
修相对跳转
<CPP>
for (auto& r : relJmps) {
    int32_t oldRel = read<int32_t>(&newBuffer[r.addrInNewBuffer]);
    uint32_t oldBase = r.addrInBuffer + 4; // 相对基址 = 立即数末尾
    uint32_t newBase = remapOldToNew(oldBase);
    uint32_t oldTarget = oldBase + oldRel;
    uint32_t newTarget = remapOldToNew(oldTarget);
    int32_t newRel = (int32_t)((int64_t)newTarget - (int64_t)newBase);
    write<int32_t>(&newBuffer[r.addrInNewBuffer], newRel);
}
这个写法比你后面手动算区间 offset 稳很多，前跳后跳都能处理。
